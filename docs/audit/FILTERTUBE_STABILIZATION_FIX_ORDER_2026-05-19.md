# FilterTube Stabilization Fix Order - 2026-05-19

Status: audit consolidation. This is not an implementation patch.

The current audit baseline is:

```text
npm run audit:runtime
tests 1391
pass 1391
fail 0
```

Those passing tests prove current behavior and current gaps. They do not make
runtime behavior changes safe by themselves. The implementation gate remains closed.
The P0 fixture wall still needs runnable positive, negative, mode,
route/surface, side-effect, and restore proof for the area being changed.

## Root Cause

The disease is split authority. Empty-install lag, false hiding, end-screen
gaps, whitelist drift, and broad DOM scans are symptoms of the same shape:

```text
profile/settings state
        |
        +--> seed endpoint predicates
        +--> JSON renderer rules
        +--> DOM fallback active checks
        +--> quick-block lifecycle
        +--> fallback menu lifecycle
        +--> learned identity fetch/map writes
        +--> background compile/cache invalidation
        +--> StateManager/import/Nanah mutation paths

Result:
  one surface may think there is no work,
  another surface may still parse, scan, hide, fetch, mutate, or save.
```

The stabilization direction is a small number of shared authorities:

```text
settings + profile + surface + route
        |
        v
compiledRuleState + endpointPolicy + lifecycleBudget + mutationAuthority
        |
        +--> JSON mutation allowed?
        +--> DOM scan allowed?
        +--> menu/quick affordance allowed?
        +--> identity harvest/fetch allowed?
        +--> hide/restore/stats side effect allowed?
        +--> storage/cache/profile mutation allowed?
```

## Five-Way Review Convergence

| Review slice | Confirmed problem | Stabilization implication |
| --- | --- | --- |
| JSON/renderers | `compactPlaylistRenderer`, modern `showSheetCommand` collaborator rosters, direct watch-card renderers, Shorts owner identity, Mix/avatar-stack identity, and shelf wrappers have leaks or false-hide risks. | Do not expand JSON rules until each renderer has blocklist, whitelist, identity-confidence, route, and sibling-visible fixtures. |
| DOM/lifecycle | Members-only and playlist selectors can target broad watch/shelf containers; quick-block and fallback menu install page-wide lifecycle work before final gates; global card selectors mix surfaces. | Add lifecycle and selector authority before deleting, broadening, or moving DOM fallback code. |
| Endpoint/network/performance | Fetch/XHR interception parses and rewrites before no-rule/disabled pass-through; endpoint matching is substring-based; fallback identity fetches and synthetic clicks need budgets. | Endpoint policy and engagement budgets must come before no-work optimization. |
| Settings/mutation/security | V3/V4 settings writes, list-mode transfer, lock/session gates, sender trust, storage invalidation, and Nanah scoped apply are split. | Simultaneous allow/block and stale-alias cleanup need mutation reports and revision authority. |
| Release/static/native | Packaging is broader than manifest truth; release claims can exceed artifact proof; native runtime sync has freshness/mirror drift gaps. | Release and app sync changes need package/claim manifests, not ad hoc website copy. |
