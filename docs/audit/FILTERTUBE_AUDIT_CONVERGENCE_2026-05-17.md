# FilterTube Audit Convergence - 2026-05-17

Status: active audit synthesis. This is not an implementation patch.

Runtime proof baseline:

```text
npm run audit:runtime
tests 1018
pass 1018
fail 0
```

The audit agents independently converged on the same root cause: FilterTube's
current instability is a split-authority problem, not a single whitelist bug.
The same settings can wake JSON interception, DOM fallback, menu affordances,
identity learning, and persistence paths through different predicates and
different lifecycle rules.

## Converged Disease

```text
settings/raw UI state
        |
        v
multiple independent "is active?" decisions
        |
        +--> seed JSON interception
        +--> filter engine mutation
        +--> DOM fallback scans/hides
        +--> quick-block/fallback menu observers
        +--> background compiler/cache
        +--> StateManager/import/Nanah persistence

Result:
  empty or incomplete rules can still cost work,
  some routes mutate when they should only harvest,
  some DOM paths remain active after UI gates say no,
  some renderer gaps leak content,
  some broad containers can hide non-matching content.
```

The intended direction is one compiled rule/lifecycle authority:

```text
settings + profile + surface + route
        |
        v
compiledRuleState + endpointPolicy + lifecycleBudget
        |
        +--> JSON may mutate? yes/no
        +--> DOM fallback may scan? yes/no
        +--> quick/menu affordances visible? yes/no
        +--> identity harvest needed? yes/no
        +--> persistence/cache revision source
```

## Ignored Capture Corpus

The ignored root-level JSON/HTML/TXT files are evidence inputs, not release
source. They should remain ignored, but every high-risk renderer/endpoint
fixture should extract a small representative fragment from them when possible.
Those root captures are also the historical source material behind
`docs/json_paths_encyclopedia.md` and `docs/youtube_renderer_inventory.md`, so
they should be used as proof material while keeping the shipped extension and
website package clean.

  playlist panel, and `/next`.


   turn on when matching artifacts, store links, and checksums exist.
