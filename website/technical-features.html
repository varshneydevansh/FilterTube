Technical Features

Under the hood: How FilterTube delivers a distraction-free YouTube experience

Anti-Flicker Technology

One of the biggest challenges with content filtering is the "flash of unwanted content" – where unwanted videos appear briefly before being hidden. FilterTube solves this with a multi-stage approach:

CSS-First Hiding

All potential elements are hidden immediately using CSS selectors that trigger as soon as YouTube loads its DOM structure.

Selective Revealing

Content that passes your filters gets the .filter-tube-visible class which makes it visible again.

/* Initial hiding rule */
ytd-video-renderer,
ytd-grid-video-renderer,
ytd-rich-item-renderer {
    display: none !important;
}

/* Selective revealing */
.filter-tube-visible {
    display: block !important;
}
Result: Zero flickering as you browse YouTube. Once content loads, you only see what you actually want to see, with no distracting elements appearing even for a millisecond.

Layout Preservation System

Simply hiding elements can break YouTube's layout, especially for complex components like Mix cards and grid layouts. FilterTube implements:

  • Horizontal Layout Preservation: Maintains side-by-side layouts for mix content using CSS flex properties
  • Grid Layout Restoration: Preserves proper grid structures on channel pages to prevent awkward layout shifts
  • Adaptive Layout Detection: Detects YouTube's layout patterns and applies appropriate fixes
function fixSearchResultsLayout() {
    // Handle Mix cards with horizontal layout
    const searchMixElements = document.querySelectorAll('.yt-lockup-view-model-wiz--horizontal');
    
    searchMixElements.forEach(mixElement => {
        if (mixElement.classList.contains('filter-tube-visible')) {
            // Force horizontal layout with flex
            mixElement.style.display = 'flex';
            mixElement.style.flexDirection = 'row';
            
            // Set proper widths for child elements
            const imageContainer = mixElement.querySelector('.content-image');
            const metadataContainer = mixElement.querySelector('.metadata');
            
            if (imageContainer) imageContainer.style.width = '50%';
            if (metadataContainer) metadataContainer.style.width = '50%';
        }
    });
    
    // Fix channel page grid layouts
    if (window.location.pathname.includes('/channel/')) {
        const gridContainers = document.querySelectorAll('#contents.ytd-rich-grid-renderer');
        
        gridContainers.forEach(grid => {
            grid.style.display = 'grid';
            grid.style.gridTemplateColumns = 'repeat(auto-fill, minmax(320px, 1fr))';
        });
    }
}

Multi-Level Channel Filtering

FilterTube provides three different methods to identify and filter channels, offering maximum flexibility and precision:

Channel Name (Partial Match)

Uses substring matching to filter channels containing the specified text anywhere in the name.

Example: "Gaming" filters "Gaming Channel" and "Pro Gaming"

@Handle (Exact Match)

Uses exact matching for channel handles starting with @, ensuring precision.

Example: "@GamingLive" only filters that exact handle

Channel ID (Exact Match)

Uses the alphanumeric channel ID for maximum precision, helpful for channels without custom handles.

Example: "channel/UC3xnGqlcL3y-GXz5N3wiTJQ"

Advantage: This multi-level approach solves common YouTube filtering problems, such as accidentally filtering too many channels when using partial text matches.

Dynamic Content Handling

YouTube loads content dynamically as you scroll. FilterTube uses a sophisticated approach to ensure all content is properly filtered, even with infinite scrolling:

  • MutationObserver System: Detects when YouTube adds new content to the DOM
  • High-Priority Processing: Special handling for critical elements like recommendations and feed videos
  • Throttled Processing: Optimized performance by batching filter operations for less important elements
  • Smart Re-filtering: When preferences change, only affected elements are processed
// Configuration for observer
const observerConfig = {
    childList: true,    // Watch for added/removed elements
    subtree: true,      // Watch the entire subtree
    attributes: true,   // Watch for attribute changes
    attributeFilter: ['class'] // Only care about class changes
};

// High-priority elements get immediate filtering
const highPrioritySelectors = [
    'ytd-rich-item-renderer',    // Home page items
    'ytd-grid-video-renderer',   // Channel page videos
    'ytd-video-renderer'         // Search results
];

// MutationObserver callback
const observerCallback = (mutations) => {
    for (const mutation of mutations) {
        if (mutation.type === 'childList') {
            // Check if high-priority elements were added
            const needsImmediateFilter = Array.from(mutation.addedNodes)
                .some(node => node.matches && highPrioritySelectors.some(
                    selector => node.matches(selector)
                ));
                
            if (needsImmediateFilter) {
                // Filter immediately
                applyFilters(filterKeywords, filterChannels);
                return;
            }
        }
    }
    
    // Otherwise throttle filtering for better performance
    if (throttleTimeout) return;
    throttleTimeout = setTimeout(() => {
        applyFilters(filterKeywords, filterChannels);
        throttleTimeout = null;
    }, 500);
};

Tab View Architecture

FilterTube provides two ways to manage your filters: a compact popup and a full-page tab view. This dual-mode architecture offers:

  • Responsive Design: Adapts to available screen space for optimal UX
  • Consistent Storage: Both views use the same storage backend, keeping filters in sync
  • Accessibility Focus: Full-page view offers better readability and more space for filter management
ADHD-Friendly: The tab view provides a more focused environment for managing filters, which can be especially helpful for users with attention differences who may find the compact popup too constraining.

Comment Filtering Engine

FilterTube offers two specialized options for handling YouTube comments:

Complete Comments Removal

Entirely removes the comments section from all videos, creating a distraction-free viewing experience.

Selective Comment Filtering

Applies your keyword filters to comments, hiding only those containing filtered terms.

Implementation details:

  • CSS-based hiding for complete comment removal (better performance)
  • DOM-based filtering for selective comment hiding
  • Dedicated mutation observer for comment-specific elements
  • URL change detection to re-apply comment filtering when navigating between videos
/* CSS for hiding all comments */
.filter-tube-hide-all-comments #comments {
    display: none !important;
    visibility: hidden !important;
}

/* JavaScript for selective comment filtering */
function filterComments(keywords) {
    document.querySelectorAll('ytd-comment-thread-renderer').forEach(comment => {
        const commentText = comment.querySelector('#content-text')?.textContent.toLowerCase();
        
        if (keywords.some(keyword => commentText.includes(keyword))) {
            comment.classList.add('filter-tube-hidden-comment');
        } else {
            comment.classList.remove('filter-tube-hidden-comment');
        }
    });
}

Privacy Architecture

FilterTube is designed with a local-first, privacy-respecting architecture:

  • Local Storage Only: All filtering rules and preferences stored exclusively on your device
  • Zero Remote Data: No analytics, telemetry, or tracking of any kind
  • Content Script Isolation: Extension operations limited to YouTube domains via manifest permissions
  • Minimal Permissions: Only requests the bare minimum browser permissions needed to function
Privacy by Design: FilterTube can't misuse your data because it doesn't collect any. All processing happens exclusively on your device, and no browsing behavior is ever transmitted.
